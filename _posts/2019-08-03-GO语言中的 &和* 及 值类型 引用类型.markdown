## 1.&和*
首先要搞懂 指针这个东西是啥。**指针**是一种数据类型，他的值是一个内存地址，根据内存地址上存值的类型，指针对应着会是 int型指针 string型指针 等，定义一个指针的方法是： var 变量名 *指针类型，如：
```
var a *int
```
此时，变量a就是一个int型指针，他只能被赋值为int型变量的内存地址，如：
```
b := 1
a = &b
```
变量a 此时的值是 0xc000058058 这种格式的内存地址，是通过&符号获取的变量b的内存地址。同样的道理，&符号获取的内存地址只能赋值给指针。

同时 变量a虽然是一个指针，但它本身也会占用一个内存块，如果你使用 &a 将会得到另一个指针，它的值是a的内存地址。
如果你想要取出一个指针指向的值，可以使用*符号，如：
```
c := *a
```
此时，变量c的结果就是 从指针a上取出的值，即 1。

## 2.数据类型-值类型和引用类型
ok，当你使用 &符号获取 array，slice，map，stuct，channel 类型的变量的指针时，你会发现他的值不是内存地址，而是引用，如：
```
d := [2]int{1,2}
e := &d
```
这个时候指针变量 e的值是 &[1,2]，而e的数据类型是 *[2]int 类型的指针，这里应该是go语言的语法糖（不确定），同时（可能是为了方便使用），array 和 stuct的指针是能够直接进行操作的，如：
```
e[0] = 2
e[1] = 3
```
e是一个指针，却无需进行 *取值，便可以直接的进行操作。而 slice，map，channel 的指针虽然同样是引用，但是，是不能进行直接操作的。原因应该是 slice，map，channel 是引用类型的，而array和stuct是 值类型。**指针**也是引用类型，引用类型的引用 大概 类似于值类型的指针的引用（如：数组指针的引用，**[2]int）也是无法直接操作的，如：
```
f := &e
f[1] = 3 //这里会报错的
```
